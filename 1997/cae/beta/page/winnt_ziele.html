Entwurfsziele von Windows NT

Windows NT wurde nicht als Weiterentwicklung eines älteren Produktes geplant. Die Systemarchitekten begannen mit einem leeren Blatt Papier un der folgenden Liste von Zielen für das neue Betriebssystem:
  1. Kompatibilität
    • Verwendung der Oberfläche von Windows
    • Unterstützung der Dateisysteme FAT ("File Allocation Table") und HPFS ("High Performance File System")
    • Lauffähigkeit von MS-DOS, WIndows, OS/2 1.x und POSIX Anwendungen
    • Herstellbarkeit der Verbindung zu den wichtigsten bereits verbreiteten Netzwerken

  2. übertragbarkeit
    • Lauffähigkeit auf CISC (Intel 386 aufwärts) RISC (MIPS R4000, DEC Alpha) Prozessoren

  3. Skalierbarkeit
    • Nutzung der Leistungsfähigkeit von symmetrischen Multiprozessorsystemen
    • Dadurch systemübergreifende Entwicklungsumgebung

  4. Sicherheit
    • Einheitliche Sicherheitsarchitektur, die die Ansprüche der US-Regierung erfüllt (Sicherheitseinstufung C2)
    • Dadurch Gewährleistung des Arbeitens mit vertraulichen Daten

  5. Verteilte Systeme (Distributed Processing)
    • Unterstützung von beliebigen Transportprotokollen (Multiple Transport Protocols)
    • Systemeigene Client/Server-Einrichtungen (High-Level Client Server Facilities), dadurch Anschluﬂ an die meisten verbreiteten Host-Systeme
    • Client/Server-Einrichtungen sind z.B. Named Pipes, nichtlokale Prozeduraufrufe (RPC-Calls) und Windows Sockets-Bibliothek

  6. Zuverlässigkeit und Robustheit
    • Verhinderung der Beschädigung des Betriebsystems und der Anwendungen durch Fehlverhalten einer Anwendung
    • Wiederherstellung von Informationen nach Abstürzen

  7. Internationalisierung
    • Einführung des im Unicode festgelegten Zeichensatzes

  8. Erweiterbarkeit
    • Modulare Architektur, d.h. möglichst feine Körnung einzelner voneinander abhängiger Systemkomponenten
